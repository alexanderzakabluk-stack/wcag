MANUAL CHECKS THAT AUTOMATION MISSES

These 22 checks require human evaluation — automated tools cannot fully assess them.

01

Text Legibility

Can all text be read comfortably at normal zoom levels? Check font size, line height, contrast and overall readability.

🔒
02

Interactive Target Size

Are all interactive elements (buttons, links, form controls) large enough for users with motor impairments? WCAG 2.2 requires 24×24px minimum.

🔒
03

Keyboard Flow

Can all interactive elements be reached and activated using only a keyboard? Check for logical tab order and no keyboard traps.

🔒
04

Visible Focus Indicator

Is the keyboard focus indicator visible and clear on every interactive element? Verify with browser zoom at 200%.

🔒
05

Skip Navigation

Does the skip-link actually work and move focus past repeated navigation? Test with screen readers (NVDA, VoiceOver).

🔒
06

Reflow at 320px

Does all content reflow to a single column at 320px viewport width without horizontal scrolling or content loss?

🔒
07

Text Resize to 200%

Can text be resized to 200% via browser settings without loss of content or functionality?

🔒
08

Zoom Support

Does the page work at 400% browser zoom? Check for overlapping elements, hidden content, and broken layouts.

🔒
09

Error Identification

Are form errors clearly identified in text, announced to screen readers, and linked to the specific field?

🔒
10

Error Recovery

Does the site provide helpful error suggestions and allow users to correct mistakes before submitting irreversible actions?

🔒
11

Timeout Warnings

Are users warned before sessions expire and given at least 20 seconds to extend? Relevant for forms and logged-in areas.

🔒
12

Moving / Blinking Content

Can all moving or auto-updating content be paused, stopped, or hidden? Verify carousels, banners, and animations.

🔒
13

Audio / Video Controls

Does all media have accessible controls (play, pause, volume)? Are controls keyboard-operable and labelled?

🔒
14

Captions Quality

Are captions accurate, synchronised, and include speaker identification and relevant sound descriptions?

🔒
15

Reading Order

Does the DOM reading order match the visual order? Test by linearising CSS and navigating with screen reader.

🔒
16

Consistent Navigation

Are repeated navigation components (menus, breadcrumbs) presented in the same order and location across all pages?

🔒
17

Consistent Labels

Are components with the same function labelled consistently across the entire site?

🔒
18

Plain Language

Is content written at a reading level appropriate for the audience? Are technical terms explained?

🔒
19

Complex Instructions

Are instructions for forms and tasks clear and not reliant on sensory characteristics (colour, shape, position) alone?

🔒
20

Cognitive Load

Is the interface simple and consistent enough for users with cognitive disabilities? Avoid unexpected changes of context.

🔒
21

Time-Based Interactions

Can users complete all tasks without time pressure? Are timed responses adjustable or removable?

🔒
22

Assistive Technology Testing

Has the site been tested with screen readers (NVDA, JAWS, VoiceOver) and other assistive technologies across key user journeys?
